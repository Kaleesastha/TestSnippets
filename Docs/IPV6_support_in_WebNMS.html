Here are our pointers in V6 IP support.

* WebNMS 5.0 support IPv6 managed nodes

* On IPv6 OS, if there is IPv6 node, Web NMS can manage it. Also on IPv6 OS if there is IPv4 node, Web NMS can manage it. So if a node supports both IPv4 and IPv6, Web NMS can manage it using IPv4 or IPv6.

* No special configurations are required to support mixed status. To manage a device in IPv6 node, you have to discover that node in IPv6 by configuring the seed.file. To configure seed.file to discover IPv6 device, refer:
http://www.webnms.com/webnms/help/developer_guide/discovery/configuring_discovery/disc_configuration.html#IPV6

* Instead of seed.file, you can also use the client "Add Node" GUI to discover IPv6 node. This UI has a separate option for providing and adding IPv6 nodes.

* you should dicover the device in IPv6 using seed.file or through add node UI. If the device is already discovered in Ipv4, then you have to delete and discover it again in IPv6 mode.

* you need not change the database/schema for this.

* You can add IPv6 device by seed.file configuration or through add node UI. In seed.file, you can give multiple IPv6 addresses of multiple devices so that all the devices will be discovered in a single run. If the device is already discovered in Ipv4, then you have to delete and discover it again in IPv6 mode.
Ensure the following to discover IPV6 nodes:
(Here we are taking ff02::1 as an example)

1. Pinging the multicast ip, ff02::1 returns a valid output (ping6 ff02::1 -I eth0)

2. Ensure that the path /usr/sbin/ping6 is present in your Linux machine. If not create a soft link as: ln -s <Real_Path_of_ping6> /usr/sbin/ping6

3. In seed.file, DISCOVER_LOCALNET has to be set as true.

4. Configure seed.file as:
<TO_DISCOVERIPV6>
<MULTICAST ADDRESS="ff02::1"/>
</TO_DISCOVERIPV6>

Now restart the server. 
If you face any issues, please get back to us with the following:

1. Output of ping6 ff02::1 -I eth0. (In windows,the command to ping IPV6 nodes is: ping -n 1 <v6_address> Eg: ping -n 1 fe80:0:0:0:868f:69ff:fe44:282d)
2. seed.file
3. Output of the command ls -latr /usr/sbin/ping6

This will help us to resolve the issue.

IPV6 Trap processing:
 * You can use trap filter to process traps from IPV6 nodes. But note that to retrieve the IPV6 address of the agent from the Snmp PDU you have to use SnmpPDU.getRemoteHost() in the trap filter.

Here are some limitations on IPV6 nodes with respect to SNMP traps :

* In SnmpV1 traps, the agent-addr field is declared as 'NetworkAddress'. Since the IPv6 Agent does not have a valid IPV4 address, it can put an unspecified address( '0.0.0.0') in agent-addr field.
Following code snippet in the trap filter retrieves the IPV6 address of the agent.
UDPProtocolOptions opt = (UDPProtocolOptions)SnmpPDU.getProtocolOptions(); 
String remoteHost = opt.getRemoteHost();

* In Snmpv2c traps, there is no field like agent-addr. But the SnmpTrapAddress OID is one of the optional varbind in Snmpv2c trap in case of proxy Agent. The type of this OID is declared as 'IpAddress'. But IPV6 Agent can not use SnmpTrapAddress OID as a varbind.

There are no specific separate tables for IPV6 in Web NMS. All data are persisted in the default Web NMS database tables only.

* Information on Discovery and Topology of IPV6 nodes is available in the following links :
http://www.webnms.com/webnms/help/developer_guide/discovery/configuring_discovery/disc_configuration.html#IPV6
http://www.webnms.com/webnms/help/developer_guide/topo/topo_modelingresourcesasmo.html#modeling_ipv6_nodes
Note that we do not have any discovery filter or trap filter example specific to IPV6.

Following are the OIDs that are queried for IPV6 discovery : 

.1.3.6.1.2.1.55.1.3.0 -  To check if the IPV6 MIB is supported in the device
.1.3.6.1.2.1.55.1.1.0  - To check if it is an IPV6 router
.1.3.6.1.2.1.55.1.11.1.5  - To get the IPV6 routing table address 
.1.3.6.1.2.1.55.1.12.1.2 - To get the IPV6Addresses defined in the ipv6NetToMediaTable
.1.3.6.1.2.1.55.1.8.1.2 - To get the IPV6 interfaces

* Web NMS does support IPv6 discovery in SNMPv1, v2c & v3 versions
