1. Introduction

                       This document explains the steps to be followed to run Web NMS server and client using  Eclipse.

2. Getting Started.  


2.1.  Creating WebNMS Project
  • Invoke the Eclipse IDE
  •  Select the menu item File > New > Project.... to open the New Project wizard.
  • Select Java Project  and click Next. In the next dialog, type "WebNMS" in the Project name.
  • Select the radio button Create project at external location. Specify the Web NMS installation directory in the Directory field. For example. if your WebNMS is installed at /home/sam/AdventNet/WebNMS/ then the path of external directory should be /home/sam/AdventNet/WebNMS/
  • Click next, to view the details about the build path.
  • Add the classes directory... using Add Class folder option. Click tthe "Create New Folder" & then in the popup provide some name (eg. nmsclasses) in the Folder name:  and then click Advanced & enable the Link to folder in the file system , then browse and provide the NMSHOME/classes directory
  • In the Order and Export tab select all and then bring the "nmsclasses" to the top.
  • Change the default output folder NMSHOME/projectclasses
  • Click finish to create the project
  • Once this done, lot of errors and warnings will be listed in the Problems tab. Approximately 488 (errors+warnings) will be displayed in the right side Problems tab.
  • In the left handside tree,  under the project the source files which have errors will be marked in red cross. Please delete those problematic files by selecting the file/directory in the tree and then deleting using the right click menu. This deletion will delete from the Product installation directory itself. If you wish to keep the following directories for your refrence, take a backup of these directories and keep it in a directory outside the NMSHOME directory structure. Then perform the following deletion
    • Delete the default_impl/client_server_communication directory
    • Delete the examples/corba/sources/ directory
    • Delete the examples/ejb_snmp directory
    • Delete the examples/provisioning/templates/scheduling/(default package)  directory
    • Delete the examples/topo_notification_registry/(default package) directory
    • Delete the genTrap/(default package) directory
    • Delete the genTrap/nmsagent_v3settings directory
  • Fix the compilation warnings by changing the compiler properties.
    • Right click on the project on the tree and select the properties.
    • Select the java Compiler.
    • On the right side pane, select "Use project settings"
    • Change warning messages to ignore for the following
      • Usage of deprecated API: (Advanced tab)
      • Unused imports  (Unused Code tab)
      • Assignment has no effects (Style tab)
      • Non-static access to static member (Style tab)
  • The README files may generate warnings, please delete them from the left side tree.
  • After fixing the errors rebuild the project. The above step of fixing the errors & warnings is important for the debugging the project later
2.2. Running the Server
  • Select the project "WebNMS" in the tree & then select the menu item Run > Run .
  • Select Java Project from the configurations list & click New
  • Provide some name to the New configuration... say server.
  • Provide the main class as "com.adventnet.launcher.nms.StartNmsJdbc"
  • Click on the Include external jars when searching for a main class.
  • Select the Arguments tab & provide the following entries.

Program Arguments:

      • APACHE_DIR <Web NMS Home>/apache TOMCAT_DIR <Web NMS Home>/apache/tomcat WEBSERVER_PORT 9090 WEBCONTAINER_PORT 8009 TOMCAT_SHUTDOWNPORT 8005 
        1. where <Web NMS Home> should be replaced by actual WebNMS Home of a installation
        2. Else, you can define a variable nmshome to have the reference to actual WEB NMS HOME, by clicking the variables button.


VM Arguments:
      • -Dresource_check="9090,8009,8005,1099" 
      • -Djava.rmi.server.codebase="http://localhost:9090/classes/ http://localhost:9090/classes/ManagementServer.jar http://localhost:9090/classes/AdventNetTftp.jar http://localhost:9090/classes/AdventNetCLI.jar http://localhost:9090/classes/AdventNetSnmp.jar http://localhost:9090/AdventNetSNMPv3/classes/ http://localhost:9090/classes/AdventNetJmxAgent.jar http://localhost:9090/classes/xmojo.jar http://localhost:9090/classes/jta.jar" -mx100m
      • -Djava.library.path=<Web NMS Home>/lib


      • At the bottom of the Arguments tab remove the selection of "Use default working space".
      • Click on the "File System" & browse to the Web NMS home & click OK.
      • In the environment tab... click New ... in the Name column give LD_LIBRARY_PATH and in the value give NMSHOME/lib:NMSHOME/apache/lib.  (not needed for Windows)
2.3. Running the Client
  • Select the project "WebNMS" in the tree & then select the menu item Run > Run .
  • Select Java Project from the configurations list & click New
  • Provide some name to the New configuration... say client.
  • Provide the main class as "com.adventnet.nms.startclient.WebNMSClient"
  • Click on the Include external jars when searching for a main class.
  • Select the Arguments tab & provide the following entries

Program Arguments:

      • hostname port yes/no
        • hostname ---> in which server running (e.g localhost or green)
        • port ---> webserver port (eg. 9090)
        • yes ---> to show log console.
      • Example entry will be like this localhost 9090 yes
      • At the bottom of the Arguments tab remove the selection of "Use default working space".
      • Click on the "File System" & browse to the Web NMS home & click OK.
