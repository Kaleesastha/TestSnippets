By default, the directories under the WebNMS Home are accessible through the browser. You can very well block their access. We have tested the following procedure to block certain directories and files. The list of directories and list of files that can be blocked and those that are not to be blocked are attached as a document

You can even block a directory at the top level alone, but permit access for the files inside it. For example, you can block access to 'logs' directory and the files contained in it as a whole or you can block only 'logs' directory at the top level and allow access selectively for particular files inside it. You may hide this by configuring the httpd.conf available under <Web NMS Home>/apache/conf/backup.

Case (1) Blocking the 'logs' directory as well as the files inside it: (configuration in httpd.conf)

<Directory "WebNMS_HOME/logs/">
AllowOverride None
Deny from all
</Directory>

This blocks the 'logs' directory. You cannot see any file list.

Case (2) - Blocking the Directory at the top level, but allowing access to all or a specific file inside it:
<Directory "WebNMS_HOME/logs/">
AllowOverride None
Deny from all
<Files *.txt (or nmserr.txt)>
Order allow,deny
Allow from all
</Files>
</Directory>

In this case http://localhost:9090/logs is not accessible but http://localhost:9090/logs/nmserr.txt will be accessible.

For further details, please refer to Apache's Document:
http://httpd.apache.org/docs-2.0/

You can allow certain user to the files can be achieved by the authentication mechanism provided by the apache, where the user has to provide a username and a password.

For further details, please refer the following Apache's Document:
http://httpd.apache.org/docs/1.3/howto/auth.html
