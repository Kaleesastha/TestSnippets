Document explaining Product upgrade from Web NMS 2.3 + SP7 to Web NMS 4.0 + SP2

This document explains the steps to be followed to ensure smooth upgrade from SP7 to SP8. The document has the following main sections.

  • Constraints for Project Upgrade.

  • Suggested Migration Path.

  • List of Customizable Files that will be over-written in SP8.

Constraints for Project Upgrade:

The following are the constraints in the Project Upgrade to Web NMS 4.0 SP2 from Web NMS 2.3 SP7.

  • The Web NMS over which the SP8 PPM will be applied should not have any structural changes (like renaming a script file, creating a Jar file for default Web NMS classes, etc). Instead of renaming/removing a script file, make a copy of the file and make the changes in the new file. This ensures that all changes pertaining to that file in SP8 are properly merged.

  • Ensure that there are no comments in the xml files (like logging_parameters.conf, Tree.xml, launcher_conf.txt). If you have any comments, remove the comment nodes and add them after the updation process is completed.

Suggested Migration path:

  • Make sure you take a backup of your customized files that will be overwritten (Please refer the table below for details).

  • Ensure that the above mentioned constraints are taken care.

  • Apply the Service Pack 8 over Web NMS 2.3 + SP 7.

  • After installing the SP8 PPM, apply the WEBSERVER PPM (will be of name WEBSERVER_Platform.ppm) that will be placed under the WebNMSHome.

  • Apply the Web NMS 4 SP2 PPM over the product.

  • For JSP and servlets, merge the changes that you have done.

  • For the conf files, if you have not modified any files that are listed in the below mentioned table, there will not be any issues as we do not overwrite the user's modifications.

List of Customizable files modified between SP7 and SP8

The following are the complete list of customizable files that are over-written by while applying the Web NMS SP8 Patch. The list includes all the files that a customer could have modified. You can omit the directories/files that you have not customized. Also, the list contains platform specific files (like .bat and .sh files). Kindly exclude those files also. The following are the list of directories for which the list has been prepared.

PS: We do not overwrite any configuration files in SP2.
