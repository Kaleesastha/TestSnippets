Scenario:3. Host 1: Primary Server Shut Down – MySQL remains running. what happens:
[Customer]: 1. To the WebNMS server on Host 2?
[WebNMS]: After ensuring that the PRIMARY is really shutdown (by getting a RMI handle & double checking with the database's primary row) the STANDBY starts as a primary for its database.

[Customer]: 2. To the WebNMS server of host 1 when it is re-started?
[WebNMS]:When the MySQL 1 is running, server1 starts as STANDBY to the existing PRIMARY server
   
Scenario:4 In Host machine 1, the MySQL database is shut down while Primary Server is running. what happens;
[Customer]: 1. To the WebNMS server on Host 2?
[WebNMS]: Server1 will shutdown & eventually Server2 will start as PRIMARY (When either the database or the server shuts down, its a failover & STANDBY will take over)

[Customer]: 2. To the WebNMS server on Host 1 when MySQL is re-started?
[WebNMS]: Again this will start as STANDBY server for the existing PRIMARY server

[Customer]: 3. If on Host1 MySQL is down for 30 minutes?
[WebNMS]: On the next database restart, only the replication happens.
If you have configured for DB_RECONNECTION & RECONNECTION_TIMEOUT is less than 30 minutes, PRIMARY will wait for DB1. (Any way 30 min. for DB_RECONNECTION_TIMEOUT is highly undesirable). If you have not specified these & then you down MySQL1 for 30 min, then server1 will go down; If you manually restart the server1, it will start as STANDBY server

Scenario:5 Host 2: standby Server Shut Down – MySQL remains running. what happens:
[Customer]: 1. To the WebNMS server on Host 1?
[WebNMS]: It will continue to be running as PRIMARY server. (MySQL 1 & 2 will be replicating). BEFailOver table will be properly cleaned up

[Customer]: 2. To the WebNMS server on Host 2 when it is re-started?
[WebNMS]: It will again start as the STANDBY server

Scenario: 6 Host 2: MySQL is shutdown while Standby Server is running. what happens:

[Customer]: 1. To the WebNMS server on Host 1?
[WebNMS]: Server1 will run properly as PRIMARY. The replication will be lost (on the database level)

[Customer]: 2. To the WebNMS server on Host 2 when MySQL is re-started?
[WebNMS]: When Server2 is manually restarted, it will be started as STANDBY.

[Customer]: 3. If on Host2 MySQL is down for 30 minutes?
[WebNMS]: Again if there is no DB_RECONNECTION  configured. it will get shutdown (Again here it is not desirable to keep the TIMEOUT as 30 min)
