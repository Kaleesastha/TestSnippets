Feature:Organizing Configuration Actions (prov_org_config_act)
-----------------------------------------------------------------------------------

Provisioning Templates are defined and stored under <WebNMS Home>/provisioning templates directory.

New functionality added for organizing Configuration Actions Templates:

    * To organize the templates and store under the appropriate sub-directories under provisioningtemplates directory.

Below mentioned methods are provided in ProvisioningAPI for getting the templatelist, template and templateparams.

    * getTemplateList(String path)

Here the template names from the specified path are returned. path should be relative to <WebNMS Home>/provisioningtemplates.

    * getTemplate(String path, String name, String params)

Here the template from the specified path is returned as string.path should be relative to <WebNMS Home>/provisioningtemplates. params specifies the template parameters for initializing the template. If there is no template parameters to be replaced, then it is given as null.

    * getTemplateParameters(String path, String name)

Here the provisioning template input parameters, which may be needed for initial population of the template present in the specified path is returned as string. Path should be relative to <WebNMS Home>/provisioningtemplates.

TestSetup:
----------

1) Unzip the file         prov_org_config_act_Validate < date of creation >.zip over <Web NMS Home>/provisioningtemplates  directory.

Before starting the WebNMS Server:

2) Run the     " before_startnms.sh/bat" present under <Web NMS Home>/provisioningtemplates/prov_org_config_act_Validate/ directory. before_startnms.sh/bat modifies your environment before starting the WebNMS server as follows:
  1. Adds the test, test10, test11, test12, test13, test14, test15, test16, test25, test42, test44, security, config directories and IPRoutingConfiguration.xml, ATM_Virtual_Connection.xml, CrossConnect.xml, ViewMOProps.xml templates to the <Web NMS Home>/provisioningtemplates directory .
  2. Removes the test13,test14 directories under the <Web NMS Home>/provisioningtemplates directory.
  3. Empties the test15,test16 directories under the <Web NMS Home>/provisioningtemplates directory.                                                      
After starting the WebNMS Server:

3) Run the     "after_startnms.sh/bat" present under <Web NMS Home>/provisioningtemplates/prov_org_config_act_Validate/ directory. after_startnms.sh/bat modifies your environment after starting the WebNMS server as follows:
  1. Removes the test10, test42, test44 directories under the <Web NMS Home>/provisioningtemplates directory.
  2. Empties the test11, test12, test25 directories under the <Web NMS Home>/provisioningtemplates directory.
  3. Adds the test14, test42 directories to the <Web NMS Home>/provisioningtemplates directory.
  4. Adds the IPRoutingConfiguration.xml template to the test15 directory, example.xml template to the test12 directory and CrossConnect1.xmltest25 directory  under the <Web NMS Home>/provisioningtemplates directory. template to the
  5. set the class path for the following JARs: crimson.jar, jaxp.jar, xalan.jar, ManagementServer.jar, AdventNetFramework.jar  and <WebNMS Home>/classes
How to Test:
------------
  1. Run the Testprogram "prov_org_config_act_tester.java" with arguments as path of the <WebNMS Home> and Host in which NMSServer is running.     eg:  java  prov_org_config_act_tester   <path of WebNMS Home >   <localhost>
  2. The Result is printed in the  "prov_org_config_act_Testresult.txt" and also the output relating to each testcase is stored in the "prov_org_config_act_Testoutput.txt".


Note:
  • The files prov_org_config_act_orgconfigacttester.java, prov_org_config_act_orgconfigacttester.class, prov_org_config_act_Testinput.txt, prov_org_config_act_Mastertestoutput.txt ,mastertemplates directory and templates directory should be in the same directory.
  • The result generated is applicable only the test environment as indicated in the above testsetup.
  •  If the user wants to change the directory structures in the <Web NMS Home>/provisioningtemplates directory or any inputs in the input file then user has to generate a masteroutput file and then he can validate the result.
