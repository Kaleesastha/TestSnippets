Database Checker

What is Database Checker?

Database Checker is a read only tool, which can be run as a stand-alone application on WebNMS database, to  check data consistency and integrity. This tool has been integrated into the database query tool (UItoDB), making it much more useful and powerful.

Motivation

As Sridhar Vembu said, "Data is everything. Therefore, Data is God". WebNMS is database centric and the integrity of database should be maintained all the time without exception. This tool is a first step in that direction. The tool checks data integrity by ensuring that various pieces of related data are consistent. For example, the status of the Managed Object should be the same as the severity level of the most recent Alert. Similarly, the status of the parent object should be consistent with the status of the children objects.

Checks Performed

Currently, this tool performs four major checks :
  1. Checks if the status of parent nodes are consistent with that of the interfaces.
  2. Checks if the status of the Managed Object is consistent with the Alert Status.
  3. Checks if the Alert Status is consistent with the most recent Event generated for the same entity.
  4. Checks if any successive duplicate events are generated for the same entity.
 

How do you Invoke the tool?

The directory structure is self explanatory. The tool can be invoked by calling the run.bat script. Please ensure that the command "java" is in the path.

How do you Customize the tool?

The source code is available in the src directory. All the checks are performed in the file ConsistencyChecker.java. In this class, there is a method called customCheck(), which is empty. This method can be used to fill in customized data integrity checks. To compile, use the compile.bat script. Please ensure that the command "javac" is in the path.
