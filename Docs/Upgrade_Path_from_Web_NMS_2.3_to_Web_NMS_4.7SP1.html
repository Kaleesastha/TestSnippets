 #113094 - Upgrade Path from Web NMS 2.3 to Web NMS 4.7- SP1


To upgrade from Web NMS 2.3 to Web NMS 4.7 -SP1 the upgradation requirements are as detailed below:

Upgrade Path

If you are on Web NMS 2.3 with or without any of its Service Packs prior to Service Pack 8, then following is the upgradation path:

Web NMS 2.3 + <any of its service pack> + Service Pack 8 + Web Server ppm + Web NMS 4 -SP2 + Web Client ppm (optional) + Web NMS 4.5 -SP1+ WebNMS 4.7- SP1.

Note: We suggest that you apply the Web Client ppm though it is optional. This is because, there is no further active development over the old HTML client. Moreover, the new Web Client provides a lot of enhanced features.

Equations

The following equations indicate the technically equivalence of the various combinations:

1. Web NMS 2.3 + SP8 + Web Server ppm = Web NMS 4
2. Web NMS 4 + SP2 + Web Client ppm = Web NMS 4.5
3. Web NMS 4.5 + SP1 = Web NMS 4.7
4. Web NMS 4.5 + SP1 (of Web NMS 4.5.0)+ SP1 (of Web NMS 4.7.0)= Web NMS 4.7- SP1

What is Web Server ppm?

This ppm consists the upgraded version of Apache and Tomcat bundled in Web NMS 4. The versions are Apache 2.0.40 and Tomcat 4.0.4. Inorder to move to higher Web NMS versions, you need to apply this Web Server ppm over SP8 of Web NMS 2.3.

What is Web Client ppm?

In Web NMS 4.5, there is a total revamp of the HTML client. The new 'Web Client' is built over Struts framework. To make this new Web Client available for those over Web NMS 4 and its Service Pack 2, this Web Client ppm must be applied. This is purely optional. For those who want to retain the old HTML client, this ppm need not be applied.
