This patch contains the following fixes/features :

  1. [Ticket ID: 3079048] : Issue while starting discovery of network from client when Auto Discovery is being disabled in seed.file. This issue has been fixed.

  2. [Ticket ID: 3079217] : When 'null' is given as the filter criteria for the owner field in the alarm custom view, the values are not retrieved properly. This has been fixed.

  3. [Ticket ID: 3079827] : Scheduler Details shows negative value for active/idle threads. This issue has been fixed.

  4. [Ticket ID: 3079800] : When a filtered alarm list is completely deleted, the entire alarm list is displayed instead of empty alarms panel. This issue has been fixed.

  5. [Ticket ID : 3079982] - database cursors exceeded
    [Ticket ID : 3079830] - Configuring Custom Log files using Log Configurator
    [Ticket ID:  3080124] - Fix for providing proper sorting in the Audit Trails List view.

  6. [Ticket ID : 3080965] - Queue size check included for Fault and Event threads in case of Multi Thread processing enabled.
    [Ticket ID : 3081942] - WebNms discoverying device with version v1 though SNMP_VERSION set as v2 for the network. This issue has been fixed.

  7. [Ticket ID : 3081898] : Customized MO Property form for displaying the properties in Others Tab and sorting the properties.

  8. [Ticket ID : 3082735] : Fix to update the frame title and label value while adding temporary custom view.

  9. [Ticket ID : 3086879] : Fix for ClassCastException when opening TL1 Craft Interface.

  10. [Ticket ID : 3088805] : Fix for updating column order into the database while saving custom view.

  11. [Ticket ID : 3088816] : Fix for saving sorted column order into the database.

  12. [Ticket ID : 3089991] : Fix for set the type of associated object for the map link while adding link through client.

  13. [Ticket ID : 3090685] : This issue is raised due to the assumption that there exist atleast one TID associated with a Gateway device during Tl1seed discovery. This was throwing a NPE due to missing TID entry, Now a change is done to allow a gateway alone to be discovered.

  14. [Ticket ID : 3090927] : Fix for sorting the image file list while clicking the browse button in the map properties client dialog.

  15. [Ticket ID : 3091151] : Fix for NullPointerException in TL1EventProcess.java, when there is no TID's under a GNE. This issue has been fixed

  16. [Ticket ID : 3091486] : NullPointerException in TL1GatewayProcess.java, when a Gateway is configured without any TID in tl1seed.file. This issue has been fixed

  17. [Ticket ID : 3090595] : TL1 autonomous messages processed multiple times. This issue has been fixed

  18. [Ticket ID : 3092273] : TL1GatewayProcess and TL1DiscProcess are initialized after the TL1EventProcess is initialized.

  19. [Ticket ID : 3091883] : During warm start an critical event is generated for TL1GatewayNode with the message "Reconnection failed". This has been fixed

  20. [Ticket ID : 3092524] : The TL1EventListener class is meant to be a singleton, but the getInstance() method is not synchronized. This has been fixed

  21. [Ticket ID : 3092960] : Parser issue arise during heavy load. This issue is fixed

  22. [Ticket ID : 3091229] : Patch to support Oracle Data-guard replication.

  1. Steps for Installing the patch over Web NMS 5.0 SP1 (DO FOLLOW THESE STEPS AS THEY ARE)

    Step 1: Use UpdateManager.bat/sh file present in <Web NMS Home>/bin directory to install the patch.
    Step 2: Now you can start the Web NMS Server

    If you want to revert the changes, please follow the steps mentioned below.

    Steps to Revert this patch (DO FOLLOW THESE STEPS AS THEY ARE)

    Step 1: Use UpdateManager.bat/sh and revert to previous version.
    Step 2: Now you can start the Web NMS Server.
