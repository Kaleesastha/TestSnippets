Introduction

This topic deals with all the default implementation classes handled by the Configuration Updater framework.

Types of Default Implementation Classes

The five different types of default implementation classes are as follows:

  1. NmsProcessesBEChanges:
      2.  This class handles the modifications of the NmsProcessesBE.conf file located in <Web NMS Home>/conf directory
  2. ServerParameterChanges:
      3. This class handles the modifications of the serverparameters.conf file located in <Web NMS Home>/conf directory
  3. SetEnvChanges:
      4. This class handles the modifications of the setEnv.bat/sh file located in <Web NMS Home> directory
  4. StartApplicationChanges:
      5. This class handles the modifications of the startApplicationClient.bat/sh file located in <Web NMS Home>/bin directory
  5. StartnmsChanges:
      6. This class handles the modifications of the startnms.bat/sh file located in <Web NMS Home>/bin directory
  6. XMLConfChanges:
      7. This class handles the modifications of all XML based (i.e., files with ".xml", ".conf" etc. extension)  files.

"Add_Process": Action command for adding a process in the NmsProcessesBE.conf file. This action command is used for adding a COMMENT, PROCESS, and ARGS in the file. 


"COMMENT": Command for commenting a process in the  NmsProcessesBE.conf file. 
"ADDITIONAL_COMMENT": This command is supportive to the "COMMENT" command. The value specified for this command is the name of the process which is to be commented.
For example,
<property 
                name="ADDITIONAL_COMMENT" 
                value="##java com.adventnet.nms.topodb.DBServer"/> 
    </property_set>

"PROCESS": Command for distinguishing the addition of a process in the file. 
"PROCESS_NAME": This command is supportive to the "PROCESS" command. The value specified for this command is the name of the process to be added. 
For example, 
<property_set name="PROCESS"> 
     <property name="PROCESS_NAME" 
       value="com.adventnet.nms.topodb.DBServer"/> 
 </property_set>

"ARGS": Command for distinguishing the addition of the arguments for a given process.You can specify as many number of arguments for a given process. 

For example, entries of arguments and their values for the process com.adventnet.nms.topodb.DBServer are specified as,

<property_set name="Insert_line" >

    value="previous line"/> 
    <property name="current" 
     value="Line to be inserted"/>

  </property_set>
</update_node>		 "Insert_line":  Action command to insert a line in-between two lines in the startApplicationClient.bat/sh file. The line to be inserted can be a classpath or default arguments. 
"previous":It is the line, which is before the inserting line. 
"current": : It is the new line which is to be inserted. 
"next":  It is the line which is after the inserting line.

<update_node  name="Insert_line" description="Inserting variable">

  </property_set>

        <FILTER className=""/>
       </property_set> 
     </property_set>
