You can make use of the Web NMS Maintenance and Support Tools to migrate the configuration files.

In case of database migration, you can take the backup of the data in the previous version & restore the same in 4.7 SP2
(If the EMS version is based on a version less that 4.7 namely 4.5 or 4.0, kindly tell us the existing WebNMS version he is using. We will instruct you accordingly)
Refer the WebNMS Developer guide for more options in this case.

If there are any DatabaseSchema changes between the EMS versions, , Kindly verify the WebNMS KBase article - To take care of Data Migration between EMS versions:

Excerpts from the article
If there are any DatabaseSchema changes between the EMS versions, the older data has to be migrated into the new format in the new DatabaseSchema. For this to be done, the implementation has to be written and invoked in the pre/post-script process of the UpdateManager tool, while applying the ".ppm".  The following methodology is suggested to achieve the needs:
  • A new database has to be created with the changed DatabaseSchema.
  • Once created, all the information present in the existing database has to be copied to the new one.
  • Once this is done, Web NMS can use this new database to get started.
The advantage is that the collected data is secure and the previous version of Web NMS can be run anytime.

The implementation should take care of the following while copying down the data into the new database.
  • The newly created database has to be clean without any data.
  • When a table is not present in the new version and present in the deployed version, then the data in the deployed version has to be inserted into another table in the new version.
  • When a table is present in both the new and the deployed versions, then the data present in the deployed version has to be read and populated in the new version.
Also note that as NAR is the final packaged output of the application, there is no mechanism to upgrade the NAR built on Web NMS 2.3 to Web NMS 4. However, the projects can be upgraded to Web NMS 4 and the application NAR can be re-created. (from KBase - Is there any tool to migrate a NAR built on Web NMS 2.3 to Web NMS 4 ?)

You can also refer the KBase article Upgrade Path from Web NMS 2.3 to Web NMS 4.7- SP1
