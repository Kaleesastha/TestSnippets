Dear Ravi

As of now, We do not have 64-bit installation for WebNMS 5.2 (Though we have it for 5.0)
Please follow the below procedure to convert your 32-bit installation into 64-bit ones.
(Once you did this in a Windows/ Linux machine, you can zip it and you can unzip it as & when you need)

1. Download Postgres-64-bit database & 64-bit JRE from the below links:

http://get.enterprisedb.com/postgresql/postgresql-9.1.4-1-windows-x64-binaries.zip - Pgsql 64-bit-Windows 
http://get.enterprisedb.com/postgresql/postgresql-9.1.3-1-linux-x64-binaries.tar.gz - 64-bit-Linux

JRE : http://www.oracle.com/technetwork/java/javase/downloads/jre6u35-downloads-1836473.html

2. Rename the existing pgsql directory (under NMS_HOME) to 32-bit-pgsql
3. Create a new directory pgsql under NMS_HOME
4. Unzip the above zip into this directory
5. Unzip this zip under NMS_HOME/pgsql 
6. Run the NMS_HOME/pgsql/initdb script and ensure that NMS_HOME/pgsql /data gets created.
7. Now start the server.
8. Ensure to have 64-bit JRE configured in setEnv.sh
9. Ensure that you have the following files as 64-bit ones
      1. files in NMS_HOME/lib directory
      2. wrapper executable under NMS_HOME
      3. NMS_HOME/conf/wrapper.conf file
If you do not have them, you can download the same from WebNMS Wiki.

10. Now restart the server. If you face any issues, please mail to WebNMS support about the same along with the logs and the below files.
      1. files in NMS_HOME/lib directory
      2. wrapper executable under NMS_HOME
      3. NMS_HOME/conf/wrapper.conf file
      4. NMS_HOME/logs

These details will help us to analyze the issue.

Thanks & regards
Venkatramanan
