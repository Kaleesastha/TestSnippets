But to know the impact of a specific text (say, if I keep a mapName as a 150-character text, what columns need to be expanded), do the following steps:

1. Reinitialize the server.
2. Start the server & client
3. Configure that particular text for a particular field (say mapName as a unique name - say testxxxx or security user as masterxyz123)
4. Stop the client & server.
5. Run $NMS_Home/bin/backup/BackupDB.bat script
6. Unzip the file - $NMS_Home/backup/BackUp_MMMDD_YYYY_HH_MM.data file (its a zip file only)
7. Open the BackUp_MMMDD_YYYY_HH_MM file (a flat file, not an XML file)
8. Search for the unique string (testxxxx or masterxyz123) - Now you know in which columns this text appear.
9. Reinitialize the server
10. In DatabaseSchema.conf or hbm.xml file, increase the width & restart the server.

We believe this meets your requirement. If there are any issues, kindly provide (attach / upload) us the following:
1. Modified DatabaseSchema.conf file & all hbm.xml files
2. Created BackUp_MMMDD_YYYY_HH_MM.data file.
3. The unique text (testxxxx or masterxyz123) you had given.

Kindly get back to us for any further clarification in this regard,
