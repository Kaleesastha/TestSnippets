Overview

 NmsProcessesBE.conf and NmsProcessesFE.conf are the main configuration files which contain the list of all the processes (also called 'modules') that get started when Web NMS Server is started. Different services offered by Web NMS such as Discovery, Topology, Fault Management, Performance Management, Security Management, SAS Server, TAS Server, TFTP Server, WebNMSAgentApp, Policy, Backup etc., are initiated and managed by their corresponding processes defined in NmsProcessesBE.conf and NmsProcessesFE.conf.

 Among the various processes present in NmsProcessesBE/FE.conf, some are mandatory for the normal functioning of Web NMS and some others are for serving specific functions.

 This document

  • lists down the processes which are mandatory for the functioning of Web NMS

  • lists the processes needed by each of the services offered by Web NMS

  • brings out the dependency between various processes/services

  • lists the processes that can be enabled/disabled on need basis

A Note on the Terminologies Used

 Throughout this document, three important terminologies have been frequently used :

  • Process : A process is an independent entity performing a specific functionality.

  • Module : The term 'Module' is synonymous with process and these terms are used interchangeably.

  • Service : A group of processes collectively perform a specific function. That specific function is known as a Service. In some cases, even a single process performs a specific service.

The cache size for user-defined objects (those persisted using UserStorageAPI) that are persisted in the database, can also be set in NmsProcessesBE.conf located in the <Web NMS Home>/conf directory.

For example, say you want to store 100 user-defined objects in memory. You can pass the argument 100 for the process com.adventnet.nms.store.DBUserStorageServer, as mentioned below :

PROCESS    com.adventnet.nms.store.DBUserStorageServer
ARGS       100
By default, a NULL value is specified, which implies that 25 user-defined objects will be stored in memory (This number is specified in the program DBUserStorageServer !) When there are no user defined objects to be persisted, this could be commented.

How to create a dependency between WebNMS processes

You can create a priority for a process as follows:

Note that every process which implements the RunProcessInterface contains the method isInitialized(). This method will return a boolean variable as "true" if the corresponding process has started completely. So you can use this method to achieve your requirement.

Let us assume, process-A ----> reading the license file and initializes some variables.
process-B ----> doing some database operation based on those variables.

In this case process-B should be started after the complete startup of the process-A. So you have to call the 'isInitialized' method of the process-A inside the process-B. If this 'isIntialized()' returns true, it indicates that the process-A has started completely. In the 'callMain()' method of the process-B, provide your implementation in such a way that it should get started only after the 'isInitialized' for process-A
returns 'true'. 

Example code snippet:
public class Process-B implements RunProcessInterface
{
        private boolean initialized = false;

        public void callMain (String[] args)
        {
                // This method is called by the NMS server during startup.

                RunProcessInterface process = (RunProcessInterface)(NmsUtil.runProcessModules).get("Process-A"); 
                /*If the process is properly started, then this will return the object reference of Process-A. Otherwise, this will return null value. If the value is null, then the Process-B should wait for some time until the initialization of Process-A. */
                while(!process.isInitialized())
                {
                        Thread.sleep(1000);
                }
                // implement your operation logic
                initialized = true; // This is very important
        }

        public boolean isInitialized ()
        {
                return initialized;
        }

        public void shutDown ()
        {
                // handle shutdown for the process
                ...
                        ...
        }

} // End of class NmsProcess
At the end of the process startup, you can call NmsUtil.webNMSModulesInitialized which will be initialized only when all the modules are initialized.
Based on your business logic, you can choose to shutdown WebNMS as follows:
try
{              
        Thread ShutdownThread=new Thread("Shutdown Thread") //NO I18N
        {
                public void run()
                {
                        try{
                                com.adventnet.nms.admin.ShutDownAPIImpl.getInstance().shutDownNMSServer(true); // Shuts down the server
                        }
                        catch(Exception e)   
                        {
                                e.printStackTrace();
                        }
                }
        };
        ShutdownThread.start();
}
catch(Exception se)
{
        System.err.println("Exception in Shuting Down the server   " +  se); //No Internationalization
}
