  1. MySQL Installation

  • Download the MySQL Source distribution into /usr/local/ directory of the machine. (Example: mysql-max-5.0.18-rc-linux-i686.tar.gz)
  • Login as root user into the machine.
  • The basic commands which you must execute to install and use a MySQL binary distribution are:
     shell> groupadd mysql
     shell> useradd -g mysql mysql
     shell> cd /usr/local
     shell> gunzip </usr/local/mysql-max-5.0.18-rc-linux-i686.tar.gz | tar xvf -
     shell> ln -s /usr/local/mysql-max-5.0.18-rc-linux-i686 mysql
     shell> cd mysql
     shell> scripts/mysql_install_db --user=mysql
     shell> chown -R root  .
     shell> chown -R mysql data
     shell> chgrp -R mysql .

  • Edit the my.cnf file available under the /etc/ directory. If the file is not available in the /etc/ directory, copy the my-small.cnf  file from the <MySQL Home>/support-files directory to /etc/ directory and rename the file as my.cnf. Add the below line under the [mysqld] section in the file. 
default-table-type=InnoDB


Note:

For more detailed MySQL installation procedure, please refer the document given below:
Use the following command to open the document:

 shell> cd /usr/local/mysql
 shell> vi INSTALL-BINARY



  1. Startup of MySQL
To start the MySQL execute the following command:

     shell> ./bin/mysqld_safe --user=mysql &

Connect to MySQL database by executing the following command as mysql root user:

     shell> ./bin/mysql -u root

  1. User Configuration
  • Create a user in MySQL database with a user name and password as per your requirement. In case of Replicated Database setup create the same user in both the databases. Password should be secured enough and should not be guessed easily. 
  • Using the below query to create user
 mysql> GRANT ALL PRIVILEGES ON *.* TO '<EMH User>'@'localhost' IDENTIFIED BY '<Password>' WITH GRANT OPTION;
 mysql> GRANT ALL PRIVILEGES ON *.* TO '<EMH User>'@'%' IDENTIFIED BY '<Password>' WITH GRANT OPTION;

The first command grants the privilege to the user on MySQL database in local host and the second command grants the privilege to the user on MySQL database in remote host.

Configure the same Username and Password while installing EMH Core Application.

  1. Replication Configuration
There will be two databases namely, Master and Slave. The DB which acts as MASTER can also act as SLAVE for another MASTER at the same time. Carryout the below mentioned procedure to configure the two way replication of MySQL databases:
  1. Enable Logging on both MySQL databases

    Edit the     my.cnf file available under the /etc/ directory. If the file is not available in the /etc/ directory, copy the my-small.cnf  file from the <MySQL Home>/support-files directory to /etc/ directory and rename the file as my.cnf
    A    dd the below lines under the [mysqld] section in the file.

   log-bin=mysql-bin
  server-id=1
  binlog-do-db=EMHDB
  binlog-ignore-db=mysql,test


Add the below line under the [mysqld] section in the my.cnf file, in both Master and Slave databases.
default-table-type=InnoDB


Example entry is given below:

[mysqld]

log-bin=mysql-bin
server-id=4
binlog-do-db=EMHDB
binlog-ignore-db=mysql,test


Ensure that the my.cnf file is read only. If the my.cnf is writable, MySQL will not read the file. You can also use the "mysqladmin" command to find out the location of the file from where MySQL reads the startup options.

The my.cnf file available under the /etc/ directory is a read only file.  For editing this file convert it into writable file by using the following command:

 shell> chmod 600 my.cnf
 
After editing the my.cnf file, convert it into read-only file by using the following command

 shell> chmod 400 my.cnf

The server-id in the MASTER server should not be the same as that of SLAVE server. For example, if the server-id of the MASTER server is 1, assign the server-id of the SLAVE server as 2.

  1. Restart the both the MySQL database servers using the following procedure: 
Stop the MySQL by executing the following command:

     shell> ./bin/mysqladmin shutdown -u root

Start the MySQL by executing the following command:

     shell> ./bin/mysqld_safe --user=mysql &

  1. Execute the following statement on both Slave databases (i.e., on both the databases), replacing the option values with the actual values relevant to your system

mysql> CHANGE MASTER TO MASTER_HOST='<master_host_name>', MASTER_USER='<replication_user_name>', MASTER_PASSWORD='<replication_password>', MASTER_LOG_FILE='<recorded_log_file_name>', MASTER_LOG_POS=<recorded_log_position>;


Notes for executing the above command:
  • In the database of the Active EMH Core Application, master_host_name should be the host address of the database of the Standby EMH Core Application.
  • In the database of the Standby EMH Core Application, master_host_name should be the host address of the database of the Active EMH Core Application.
  • If the master_host_name is not configured in the DNS, use the IP Address of the master host.
  • Fill the value recorded_log_file_name of MASTER_LOG_FILE with the File attribute value and recorded_log_position of MASTER_LOG_POS with the Position attribute value of the Master status. (Default value for MASTER_LOG_POS should be 1)
  • To get the value of File and Position attributes carryout the procedure below:  
    • For the MySQL configuration in Active EMH Core machine, execute the following MySQL query in the Standby EMH Core machine and get the value of File and Position attributes.
    • For the MySQL configuration in Standby EMH Core machine, execute the following MySQL query in the Active EMH Core machine and get the value of File and Position attributes.
    • Execute the below query to get the value of File and Position attributes in the Master status:
 mysql> Show Master status;

+------------------+----------+--------------+------------------+
| File             | Position | Binlog_Do_DB | Binlog_Ignore_DB |
+------------------+----------+--------------+------------------+
| mysql-bin.000003 |       98 | emhdb        | mysql,test       |
+------------------+----------+--------------+------------------+
1 row in set (0.00 sec)
  1. Execute the below query on both the Slave databases 
 mysql> start slave;
  1. You can execute the following queries to know the status of the Master and Slave servers. 
 mysql> Show Master status;
 mysql> Show Slave status;

  1. SSH Configuration

Note: This configuration is required only for performing EMH software and state management operations. Other normal EMH oprations will not be affected if this configuration is not done.
 
For EMH software and state management operations to work, the EMH Core and Mediation servers should be configured to operate via SSH public key authentication. The following are the steps to configure the Core and Mediation servers.
  • Between the EMH Core and EMH Mediation, the EMH Core server will act as the SSH Client and the EMH Mediation server will act as the SSH Server.
  • Between the EMH Core servers, each server will act as both the client and server for the other.
  • Check if both the client and server machines are using OpenSSH. This can be verified by issuing the command
    $ssh -V
    The output will contain OpenSSH followed by other information.
SSH client machine configuration:

Perform the following steps in the client machine from the user home directory to generate the RSA keys. Note that this should be the user in which EMH is being executed.
  1. client$ mkdir ~/.ssh
  2. client$ chmod 700 ~/.ssh
  3. client$ ssh-keygen -q -f ~/.ssh/id_rsa -t rsa
    1. Enter passphrase (empty for no passphrase):
    2. Enter same passphrase again:
  4. FTP the .ssh/id_rsa.pub file to the server machine.
After performing the above steps, the .ssh/ directory will contain two files, id_rsa and id_rsa.pub.
SSH server machine configuration:

Perform the following steps in the server machine from the user home directory to copy the public key generated in the client. Note that this should be the user in which EMH is being executed.
  1. server$ mkdir ~/.ssh
  2. server$ chmod 700 ~/.ssh
  3. server$ cat ~/id_rsa.pub >> ~/.ssh/authorized_keys
  4. server$ chmod 600 ~/.ssh/authorized_keys
  5. server$ rm ~/id_rsa.pub
Test the configuration by performing a SSH from the client machine to the server machine. You will be prompted for the "passphrase". The passphrase that was used when generating the keys should be given.

The above operations should be performed for the following set of configurations:
  1. EMH Core servers as SSH client and EMH Mediation servers as the SSH server.
  2. EMH Core active server as SSH client and EMH Core standby server as SSH server.
  3. EMH Core standby server as SSH client and EMH Core active server as SSH server.

  1. EMH Data Directory Configuration

EMH Core and EMH Mediation require data directories for persisting EMH data.

EMH Core

Create following directories in the EMH Core machine:
  • /data/emh
  • /data/PM
Change the directory and recursive sub-directories permissions of "/data" directory for all users using the command "chmod -R 777 /data".

Note :

During runtime EMH Core will create sub-directories under "/data/emh" directory. All the new sub-directories created, should have read/write permissions for the following users:
  • The FTP User - used by MCCDO to transfer file from/to EMH Core.
  • The FTP User - used by CAGE to transfer file from/to EMH Core.
  • The FTP User - used by EMH Mediation to transfer file from/to EMH Core.
 

EMH Mediation

Create following directory in the EMH Mediation machine:
  • /data/emh
  • /data/emh/devicefiles
Change the directory and recursive sub-directories permissions of "/data" directory for all users using the command "chmod -R 777 /data".

Note :

During runtime EMH Mediation will create some sub-directories under "/data/emh" directory. All the new sub-directories created, should have read/write permissions for the following users:
  • The FTP User - used by MCCDO to transfer file from/to EMH Mediation.
  • The FTP User - used by CAGE to transfer file from/to EMH Mediation.

  1. Installation of EMH Server

    1. Installing EMH Core

  • If previous EMH Core Installation is present in the current directory and you want install the new software load of EMH Core in the same directory, shut down the existing EMH Core application using <EMH Core Home>/bin/shutdownCore.sh script. 
  • Download the EMH Core bin file. The file name will be like <EMH_Core_010_0624_1.bin>. 
  • Change the file permission by using command  "chmod 755 EMH_Core_010_0624_1.bin"
  • Execute the bin file. (Example: "./EMH_Core_010_0624_1.bin")
  • By default the application will be installed in the current directory, where the build is available. 
  • This will install the Core software in the <CURRENT DIRECTORY>/Core/ directory.
  • Now the EMH Core installation is complete.
Note 1:  
The EMH Core and Mediation Applications should be installed in different machines.

Note 2:  
For Failover setup, both Primary and Standby EMH Core server machines should be configured in DNS.

  1. Start EMH Application

  1. Start the EMH Mediation Application
    • Execute startMediation.sh file available in the <EMH Mediation Home>/bin directory.  
      (Example: sh       startMediation.sh)
    • When you start the EMH Mediation Application, first time after the installation, EMH Installation script will try to get value using EMH Control script i.e., "/opt/ipbsc/bin/emh_control".
      If EMH Installation script is not able to get the value from the EMH Control script, it will try to get from the following environment files:
      • /etc/emhp/emhp_dynamic_env
      • /etc/emhp/emhp_static_env
Even in the environment files, if EMH Installation script is not able to get the values, you will be prompted to configure the following parameters:
  • Login Details for FTP of EMH:
    • Enter value for EMH_FTP_USER - the user of FTP account in EMH. EMH Mediation uses this user to transfer files to EMH Core.  
    • Enter value for EMH_FTP_PASSWD -  the password of the user of FTP account in EMH. EMH Mediation uses this password to transfer files to EMH Core.
    •  Now the EMH Mediation Application will get started. 
Note: Start the EMH Mediation server as a Linux machine Admin privileges user.
    1. Start the EMH Core Application
    • Execute startCore.sh file available in the <EMH Core Home>/bin directory.
      (Example: sh startCore.sh)
    • When you start the EMH Core Application, first time after the installation, EMH Installation script will try to get value using EMH Control script i.e., "/opt/ipbsc/bin/emh_control".
      If EMH Installation script is not able to get the value from the EMH Control script, it will try to get from the following environment files:
      • /etc/emhp/emhp_dynamic_env
      • /etc/emhp/emhp_static_env
      Even in the environment files, if EMH Installation script is not able to get the values, you will be prompted to configure the following parameters:
    • EMH Core Application parameters:
      • Enter value for EMH_VIRTUAL_IP - the Virtual IP Address of the core server (if Virtual IP Address is not configured, use the physical IP Address of the machine)
    • MySQL database parameters:
      • Enter value for EMH_MYSQL_HOME - the location of MySQL installation directory from which it is running as a service 
      • Enter value for EMH_MYSQL_USER - the user you have created during the Database Setup procedure. EMH uses this user to access MySQL database  
      • Enter value for EMH_MYSQL_PASSWD - the password of the user you have created during the Database Setup procedure. EMH use this password to access MySQL database 
    • Call Trace PC parameters:
      • Enter value for EMH_CALL_TRACE_PC_IP - the IP Address of the Call Trace Application machine
    • EMH parameters:
      • Enter value for EMH_MATE_IP - the IP Address of the Standby Core server 
      • Enter value for EMH_AEMS_IP - the IP Address of the AEMS
      • Enter value for EMH_FRONT_PANEL_IP - the IP Address of the EMH Front Panel
    • Login Details for FTP of AEMS:
      • Enter value for EMH_AEMS_FTP_USER - the user of FTP account in AEMS. EMH uses this user to transfer files to AEMS. 
      • Enter value for EMH_AEMS_FTP_PASSWD -  the password of the user of FTP account in AEMS. EMH use this password to transfer files to AEMS.
    • Login Details for FTP of EMH:
      • Enter value for EMH_FTP_USER - the user of FTP account in EMH. EMH Core uses this user to transfer files to Standby EMH Core. 
      • Enter value for EMH_FTP_PASSWD -  the password of the user of FTP account in EMH. EMH Core uses this password to transfer files to Standby EMH Core.
    • Login Details for SSH of EMH:
      • Enter value for EMH_SSH_USER - the user of SSH Telnet account in EMH. Other Applications use this user to SSH Telnet to EMH Core and vice versa. 
      • Enter value for EMH_SSH_PASSWD -  the password of the user of SSH Telnet account in EMH. Other Applications use this user to SSH Telnet to EMH Core and vice versa.
    • EMH OM Gateway parameter:
      • Enter value for EMH_OM_DEFAULT_GATEWAY - the IP Address of the EMH OM Gateway 
    •  Now the EMH Core Application will get started.

If the emh_control platform script is not present, EMH will not be able to bind/unbind the logical IP address. In this scenario, the following print will be seen in the Server Startup console.

Configuring Virtual IP Address in Linux

Using the following command, you can configure Virtual IPs in the Linux machine at run-time without re-booting the machine  

ifconfig eth0:21 172.19.1.2 netmask 255.255.255.0 up

The interface number must be unique for each interface (for example eth0:21).

If the EMH Core and Mediation servers are installed in different networks, then add the route in the EMH Core machine by executing the following command:
route add -net <IP Address of the Network> netmask <mask value> gw <IP Address of the Core Server>

(Example: route add -net 172.19.1.0 netmask 255.255.255.0 gw 192.168.118.197)

Use the PING <Virtual IP Address of Mediation server> command from the EMH Core machine to confirm the connection between the machines.


The same interfaces can be made down by using the following command.  

ifconfig eth0:21 172.19.1.2 netmask 255.255.255.0 down

Note:
  • These commands have to be executed with the super user privilege.
  • The configurations will not be available when you re-start the machine.
