We have herewith attached the Trap Tool (as TrapTool.zip) which helps you to test the trap rate.
Configure the shell script (the home of WebNMS classes) & run the trap tool.
The image (TrapTool.JPG in the zip) will help you to configure the values.
We have also got a tuning doc to this tool (which tells about the WebNMS changes to do).

We would suggest you to use AdventNet Simulation toolkit to simulate nodes.
The evaluation version is available at: http://www.webnms.com/simulator/download_confirm.html
Once you are satisfied with the evaluation version, you can buy the same & use it for your testing.

You can run the script as follows:

When you have a WebNMS in a test machine (in which DISPLAY is available) you can specify the <NMS_Home>/classes in the NMS_CLASSES variable & can run the script.

If not, you can ftp to the wind river machine (in which WebNMS is installed) & you can download the AdventNetSnmp.jar & SNMPDebugger.jar from /opt/motorola/webnms/classes/& modify the CLASS_PATH varible accordingly (for these 2 jars)
Also specify the Java home (in the script, the WebNMS-bundled JRE is referred.) & then invoke the script.
A UI will be invoked.
You can give the server machine in the Hostname variable & then start sending traps. Kindly get back to us for any further clarification in this regard.
