Hi Anoop,

Please do the following steps which will help you to migrate your data:

1. With NMS_HOME/bin/backup/BackupDB.sh, backup your existing 4.7 Sybase data
2. Create a new MySQL database and start the MySQL server.
3. Using NMS_HOME/bin/backup/RestoreBackUp.sh, restore your data.
(Please  check this link too for WebNMS backup)

Now you can use the WebNMS migration procedure to migrate this database to 5.x.
Once you successfully migrate the data to 5.x, you can backup the database in MySQL & restore the same in postgresql.
Ensure to execute the following query in postgres prompt before restoration in postgres:
UPDATE pg_cast SET castcontext = 'i' WHERE oid IN (SELECT c.oid FROM pg_cast c inner join pg_type src ON src.oid = c.castsource inner join pg_type tgt ON tgt.oid = c.casttarget WHERE src.typname LIKE 'int%' AND tgt.typname LIKE 'bool%') 

If you face any issues, please get back to us.

Hi Ravikiran,

Please download the 5.2SP1 (Linux / Windows) 64-bit zips from here.
Go through the release notes to know what is new in 5.x.

The migration guide (which talks about Server API Changes, Java Client API Changes, Database Schema Changes, Functional Changes,Studio Project Upgrade, Packaging Changes, Process of Migration) is available at: http://www.webnms.com/webnms/help/migration_guide/index.html

The DB Migration tool is used to migrate the existing 4.7 version database to schema of Web NMS 5. The migration process include modification of existing tables such as addition of column, removal of column etc, creation of new tables and removal of unwanted tables.

Download the DBMigration.jar and DB_Migration.html (guideline document for database migration) from here
Place this jar file in the classpath and execute the class from <WebNMS_HOME> directory of Web NMS 4.7 SP3/4.7 SP4.

"java util.DBMigration".

This will generate a file by name "script_mysql" in the same directory. Kindly execute the script file as mentioned in the guidelines document.

Note: As a safety measure, take a backup of the database using Backup.sh/bat file available under <WebNMS_HOME>/bin/backup directory before the start of migration process.
Please follow the below steps to start this build.

1. Unzip this in a Linux / Windows machine and unzip this zip under NMS_HOME/classes.
2. Note down the JRE-6 / JRE-7 (preferably) directory in your machine and configure the same in JAVA_HOME in setEnv.sh
3. Start the server and choose evaluation mode (Your existing 4.7 license can also be applied).

If you face any issues, please mail to WebNMS support about the same along with the logs and the below files.
      1. files in NMS_HOME/lib directory
      2. wrapper executable under NMS_HOME
      3. NMS_HOME/conf/wrapper.conf file
      4. NMS_HOME/logs

You can also know about the performance advantages by seeing performance guide which shows the comparison between 4.7 & 5.2 WebNMS.
(We can have up to 100 users when there are adequate no. of front end servers - one BE can support 15-20 java clients & hence you will need 1BE + 5 FEs to support 100 users).

In addition, please go through the WebNMS data sheet as well which shows the number of devices supported.

We believe this answers your query. Please get back to us for any further clarification in this regard.
