Password length is restricted because of DB restrictions. While we write the password in database, it is encrypted & written. This increases the length & to be on the safer side we had limited the password length.

But by following the procedure detailed below, you can increase the password character length.

JavaUI
Security Administration is a Builder Project. In that, you can find the files AdUserWiz.java & ChangePwd.java under /projects/SecurityAdministration/Source.

You need to modify the lines ["if(passw.length() > 8)"] that checks for the password length. Instead of 8, you can put some higher value. After making the change, compile the files and update the jar AuthMain.jar which is under NMS_HOME/NetMonitor/build directory.

Also ensure to modify ChangePwd.java, the following code snippet.(instead of 8 modify it to your requirement)
oldPassword.setDocument(new LimitDocument(8));

Please note that it is advisable to keep the length less than 20 characters in order to make the encrypted password length within the limit of the column width specified in the DatabaseSchema.conf file.

WebUI
In <NMS_HOME>\webclient\admin\jsp\AddUser.jsp
search for the format: webclient.admin.adduser.password &  webclient.admin.adduser.retypepassword and change the corresponding textbox's length as the desired value.

In <NMS_HOME>\webclient\common\jsp\Login.jsp search for the format
webclient.login.password and change the maxlength accordingly.

In <NMS_HOME>\webclient\admin\jsp\ModifyUserProfile.jsp search for the following formats and change the maxlength accordingly.
webclient.admin.modifyuserprofile.changepassword
webclient.admin.modifyuserprofile.newpassword
webclient.admin.modifyuserprofile.retypepassword

We believe that the attached images jsp1.JPG & ModifyUserProfilejsp.JPG would help to achive your requirement.

Ensure to run <NMS_HOME>\bin\compileJSP.bat  after you do all these changes.

P.S In case, you want more than that limit, please increase the column width for PASSWORD in UserPasswordTable in <NMS_HOME>\conf\DatabaseSchema.conf for. If you do this, you need to reinitialize the Web NMS and start it for effecting the change.

We had tested with random values (from the acceptable set of characters) for the password.
We found that for a given password length x the encryped password could be of maximum length 2x.
So if you decide that the maximum password length would be 50 kindly set the PASSWORD column width of UserPasswordTable to 100.
We believe that this would satisfy your requirement. Kindly get back to us for any further details in this regard.
