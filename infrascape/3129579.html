On receiving traps, please note that by default trap port is 162 and this port will be opened & listening only when the WebNMS server is started by super user (root).
Please follow the below steps to check whether traps are received in WebNMS.

1. Before starting the server execute: netstat -an | grep 162 | grep -i "listen" . This should return NO results

2. Start the server. Once the server is completely initialised, execute the above command again - It should return as follows:
tcp46      0      0  *.162                 *.*                    LISTEN 
(Or atleast an output which has 162 & LISTEN in the same line)

3. Ensure that you have wireshark tool in the machine (a packet sniffer).
If its not present, please install it as explained at: bit.ly/wiresharklinux (An easy step - just 1/2 commands will do)

4. Now login as super user in the machine and type wireshark in the console, which will open the wireshark tool.

Now You can filter the packets to 162 port alone (udp.port eq 162 as explained here / or as given here)

Now you can verify that the trap packets are received by the WebNMS server or not.
