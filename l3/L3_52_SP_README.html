This patch contains the following fixes/features :

    [Ticket ID : 3085654 ]  Moved all the below 6 patches given over WebNMS5sp1 (WEBNMS_5010_L3_COMM_BRANCH) to WebNMS5.2 (WEBNMS_5020_L3_COMM_BRANCH)

  1. [Ticket ID : 3072921 ]  Support for scope zone IPv6 addresses in the SNMP APIs of Web NMS has been provided.

  2. [Ticket ID : 3071200] Encryption of auth and priv passwords in the USERTABLE (column size of AUTHPASSWORD and PRIVPASSWORD are to be changed to 255 for the USERTABLE, from the default 50).

  3. [Ticket ID : 10189694] The ArrayIndexOutOfBoundsException got while decoding in LeafSyntax.decodeInstanceString() is fixed.

  4. [Ticket ID : 3074304] The following 2 issues are fixed:
    All boolean values were restored     as true.
    When column name of tables contain keywords (defined in MySQL), tables were not properly restored. This issue is fixed.

  5. [Ticket ID : 3075503] The StatusPoll MIN Time is updated unnecessarily on every statuspolling. This causes the statuspoll to stop after some time when the current time is always less than the MIN Time. This is fixed..

  6. [Ticket ID : 3075503] Fix to avoid statuspolling stopped.While getting min value of ObjectSchedulerRUNNABLE, the where condition is removed in the DB query. This is not required due to some sometimes the min variable gets past the min value of DB and due to this the status polling never happens for that device.

  7. [Ticket ID : 3095242] One of the variable(queryString) is declared as a global variable, which causes the UnKnownEngineID issue. Now we have changed the variable scope to local. This fixes the issue.

  8. [Ticket ID : 3095563] : The issue in snmp retry and timeout getting overwritten from other target instances is fixed.

  9. [Ticket ID : 3099504]  : Postgres was not starting automatically when WebNMS is installed in a space sepatated directory. This issue is fixed

  10. [Ticket ID : 3103689]  : Support provided for BigInteger type.

  11. [Ticket ID : 3075503] : When there is a trap/Alert storm for an MO, the status is not propagated properly. Fixing this issue

  12. [Ticket ID : 3102143] - MibOperations.createVariableBinding Exception fixed


    Steps for Installing the patch over Web NMS 5.2 (DO FOLLOW THESE STEPS AS THEY ARE)

    Step 1: Use UpdateManager.bat/sh file present in <Web NMS Home>/bin directory  to install the patch.
    Step 2: Now you can start the Web NMS Server

    If you want to revert the changes, please follow the steps mentioned below.

    Steps to Revert this patch (DO FOLLOW THESE STEPS AS THEY ARE)

    Step 1: Use UpdateManager.bat/sh and revert to previous version.
    Step 2: Now you can start the Web NMS Server.
