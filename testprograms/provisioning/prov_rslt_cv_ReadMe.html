Feature: ProvisionResult Custom View
-----------------------------------------------------

Functionality:
--------------

Custom view support has  been provided for ProvisionResult object. Using the custom view, user can perform the searching, sorting and grouping operations.To get the required data( Provision Result based on some criteria), the following APIs have to be used.

ViewCriteria:
This is used to specify the criteria information based on which the data have to be fetched from the DB.

ProvisionResultSessionBean:
This is the CustomSessionBean for ProvisionResult CustomView functionality. This API will be used to fetch the ProvisionResult info in the form of 'ViewData' from the DB based on the criteria specified.


TestSetup:
-----------

1) Copy the files prov_rslt_cv.jsp,  prov_rslt_cv_Testinput.xml and prov_rslt_cv_XmlReader.java into <WebNMS Home>/jsp directory.

2) Compile the AF_14_XmlReader.java  in the test folder so that its destination directory is <WebNMS Home>/WEB-INF/classes.
    javac -d <WebNMS Home>/WEB-INF/classes prov_rslt_cv_XmlReader.java

3) SetUp required for enabling the dynamic compilation of JSP Files:
    The steps you must follow to enable dynamic compilation of a JSP file are as follows:

    The below given two steps must be executed by the users of the Windows 32 operating systems, only. They are not applicable for Linux and Solaris operating system users.
  • Set Absolute Path for Web NMS in setEnv.bat File:You must set the absolute path of location of Web NMS for the "NMS_HOME" variable in setEnv.bat file located in <Web NMS Home> directory, as shown below,
                    example: set NMS_HOME = C:\AdventNet\Web NMS
  • Add Mandatory "JARS" in Apache directory: The "manifest.inf" file of AdventNetOPExtn.jar contains the list of mandatory jars required for starting Web NMS. For dynamic compilation of your files, you must place all the jars listed in the manifest.inf file, in the <Web NMS Home>/apache/tomcat/common/lib directory.
    The following steps are mandatory for both the users of Linux and Windows 32 operating systems
  • Set "tools.jar" in System Classpath:  You must set the classpath of  the "<JDK_HOME>/lib/tools.jar" of the JDK (wherever installed) in the startnms.bat/sh file located in <Web NMS Home>/bin directory.
          Example Entry in startnms.bat/sh File:
                 In Windows OS:    set CLASSPATH=D:/jdk1.4.2/lib/tools.jar;%CLASSPATH% in startnms.bat file
               In Linux OS:          export CLASSPATH=/usr/java/jdk1.4.2/lib/tools.jar:$CLASSPATH in startnms.sh file
  • Modifying a Parameter Value in web.xml:  In the web.xml file present in <Web NMS Home>/apache/tomcat/conf directory, for the JspServlet, set the value as true for the paramater development. The example entry is as shown below:
         < style="font-family: times new roman,times,serif;">            <servlet>
        < style="font-family: times new roman,times,serif;">            <servlet-name>jsp</servlet-name>
        < style="font-family: times new roman,times,serif;">            <servlet-class>org.apache.jasper.servlet.JspServlet</servlet-class>
            <init-param>
            <param-name>development</param-name> <param-value>true</param-value>
            </init-param>
                .......
                .......
            </servlet>

Refer below mentioned link for enabling the dynamic compilation of JSP Files:

How to Test:
-------------
1) Login to the WebNMS Webclient using http://localhost:9090/ in your Browser.

2) Run the JSP file  prov_rslt_cv.jsp in your WebNMS-WebClient with the test case number as input.
    i.e: http://localhost:9090/jsp/prov_rslt_cv.jsp?testnumber=tc001
    Where "tc001"  refers to the testcase number in the testplan.

3) The  output will be displayed in the browser so that the user can verify the result with the required result mentioned in the testplan.

4) Repeat  steps 2 & 3  by changing the Testcase ID. 


Note:

a) All the inputs to the JSP file are stored in AF_14_Testinput.xml file with each element representing each Testcase input.

b) If the user wants to change any inputs he can change the appropriate values in the respective element of AF_14_Testinput.xml.

b) Test case number 52 is captured  while running all the test cases as the  ProvisionResult custom session bean is removed after viewing the sorting criteria.

d) Before testing, populate the provisionresult table with sufficient data.
