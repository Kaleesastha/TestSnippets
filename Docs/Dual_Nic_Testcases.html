TEST  CASES  FOR  BE FAILOVER  CLIENT  MODULE

Abrrevations Used :

1. SF    -  Server Frame Work

2. BEP  -  Primary Server

3. BES  -  Standby Server.

4. FOV   -  Fail Over

5. SER       -  Server.

6FE         -   Front End Server

 

Initial Steps to be followed:

1. Start the BEP server (the machine in which Dual NIC mode is enabled) in machine1, with database in the same machine

2. Start the BES server in machine2 with the DB in machine1

3. Start the FE server with BE_HOST as BEP in machine 3

4. Now connect the browser client to FE sever.

5. After the start up of client, just go across all the nodes and check for the normal client functionality.

6. Stop the BEP server.
 

Assumption:   It is assumed that all the test cases given below are done with the client started before Fail Over unless otherwise mentioned.
 
        TEST CASE 
      DESCRIPTION		         TEST CASE 
      PROCEDURE		   EXPECTED RESULT           SEVERITY           REMARKS
Test to check for the client(connected after FOV) startup after FOV. After failover takes place, start a browser client to the FE server. The client shoul get started in the same way and shoul work in the normal way as before failover without any problem.   CRITICAL-CT
Test to check for Mib Manager after FOV.  Befor FOV, from the client, do a GET variable through the Mib Manager and after FOV, through the same client do the GET variable for the same OID and check the response. The response should be shown as before and there shouldn't be any requesr timed out or any exceptions in client logs, (assumed that the agent is Up in that device).   CRITICAL
