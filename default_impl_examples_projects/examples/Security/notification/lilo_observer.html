Example Overview

Example name : LoginLogoutObserver.java
Location : <Web NMS Home>/examples/Security/liloNotification directory

This example explains the login logout observer functionality in WebNms. The actions specified in the handleClientConnection() method of LoginLogoutObserver.java (located in <Web NMS Home>/examples//Security/liloNotification directory) will be performed when there is a client login or logout and also in case of login failure.

Prerequisite

Prior to executing this example class it is imperative of the user to write the handleClientConnection() method implementing the ClientConnectionObserver class. This method will have all the actions that are to be performed on user login,login-failure or logout. The action may be a simple Print command which prints details of the notification in the console or into a file. For more information on writing the client Connection Observer please refer to the documentation on LILO Observers under User Login and Logout Notifications section in Security management. 

The interface used in this example is ClientConnectionObserver

Methods used in this example are listed below. These methods will return a boolean "true" if the observer is registered for the updates.

  • registerClientConnectionObserver


This method shall be called when a user login and logout of the server via application client, applet client, webstart client and web client.

  • handleClientConnection

 Note : The source code will have comment statements in it which will explain to you the different segments of the program.

 

Integration with Web NMS

The steps given below will help you in integrating this example with Web NMS. If any of the steps given below are not followed properly, then the example might not function in a desired manner.

  • Set the <Web NMS Home>/classes/NmsServerClasses.jar directory to classpath

  • Compile the LoginLogoutObserver.java source file [ javac -d <Web NMS Home>/classes liloObserver.java]

  • Ensure that the class file is placed in the <Web NMS Home>/classes directory after compilation, in line with the package name specified in the program

  • Create a stub for the LoginLogoutObserver.class file

  • Now start the Web NMS Server and wait till it successfully starts up

  • Execute the program

Sample Output

On executing the class file, it will register as ClientConnectionObserver with GenericBEApi. “Successfully registered with the observer would be printed” in console. Whenever a client login,logsout or there is a login failure the handleClientConnection method will be called. The type of notification and the properties for the same shall be printed.
