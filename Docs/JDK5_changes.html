When using JDK 1.5 version along with Studio tools in the following exceptions are displayed in the console when using WebNMS IDEs:
java.io.FileNotFoundException: C:\WebNMS\Testing\AdventNet\WebNMS\StudioTools\Studio\projects\<Tutorial_Name>\config\Packager.xml
(The requested operation cannot be performed on a file with a user-mapped section open)
at java.io.FileOutputStream.open(Native Method)
...
To overcome this issue, xalan.jar, crimson.jar  and jaxp.jar classes of Java 1.4 should be used. To achieve this, the Endorsed Standards Override Mechanism should be followed.

To implement this,
1. Copy xalan.jar, crimson.jar, and jaxp.jar to a folder, for example "endorsed_nms" under <NMS_Home>/classes folder.
2. In the <NMS Home>\StudioTools\Studio\bin\startWebNMSStudio.bat  file, add the below content.
-Djava.endorsed.dirs=%WEBNMSHOME%\classes\endorsed_nms

For example, after adding the above content, the JAVA_CMD entry would be:

set JAVA_CMD="%JAVA_HOME%\bin\java" -Xmx200M -Djava.endorsed.dirs=%WEBNMSHOME%\classes\endorsed_nms
The above procedure must be used to overcome the same issue in other IDEs such as Configuration Wizard, Chassis Wizard also.

Als refer Changes needed for Chassis Wizard, General Changes
