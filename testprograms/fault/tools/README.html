TestAPI.java

RMI Testing Tool based on UI. Basically this lists all the methods in TrapAPI,EventAPI,AlertAPI,EventFilterAPI,EventParserAPI,AlertFilterAPI and TrapAPI.

You have to enter the input parameters in textbox. Properties are separated by &.

Usage: java TestAPI RMIPort SecureFlag
 
 
S.No Arguments Description Values
1 RMIPort RMI Port of WebNMS Server int
2 SecureFlag Secured Server Mode or not true / false

back
 
 
AddEvent.java

Standalone tool which adds the specified number of Events into the system using EventAPI.addEvent() method.

Usage: java AddEvent NumberofEvents hostName
 
 
S.No Arguments Description Values
1 NumberOfEvents Total number of Events to be added int
2 hostName Name of the machine where WebNMS is running String

back
 
 
MultiSendTrap.java

Standalone tool which can be used to send traps at the specified rate.

Usage: java MultiSendTrap PeerName PortNumber No_Of_Traps TimeInterval(milliseconds) GT ST Enterprise printCount noOfThreads
 
 
S.No Arguments Description Values
1 PeerName Name of machine where NMS server is running String
2 PortNumber TrapPort of WebNMS int
3 NoOfTraps Total number of traps which needs to be send in single thread int
4 TimeInterval sleep time between two successive traps in millisecs
5 GT Generic Type of V1 trap int
6 ST SpecificType of V1 trap int
7 Enterprise Enterprise value of Snmp V1 trap String
8 printCount Number of traps once in which time will be printed int
9 noOfThreads Number of Threads which send traps int

Example:
NoOfTraps = 100
noOfThreads = 10
TimeInterval=1
Then totally 1000 Traps will be sent by 10 different traps. Difference in time between each thread is 1 millisec.

final.jar

This jar file is a Cache tool. It enables to do Server Cache & Transaction testcases.
As the transaction cases are automated you could use the tool to check client
side transaction effect. It is a bean builder project.
It also consists of a Readme.txt that contains all the infomation.

Steps:

  • Copy all class files under cache/classes directory of the tool into classes directory of the Web NMS server or have it server classpath.
  • Have AdventNetUtils.jar in server classpath.
  • Include the below process entry in NmsProcessBE.conf
      • #java com.adventnet.nms.karthi.cachetest.CacheProcess
      PROCESS    com.adventnet.nms.karthi.cachetest.CacheProcess
      ARGS      NULL
  • Restart the server.
  • A UI will pop up. Work with the options available
back
 
 
EventQueSize.java

This implementation will check the que size periodicaly for given interval and given stretch of time.
You could add events using performance simulator tool.
Events are stored into a buffer periodically and at the end of Time Period, dumped into a file.

Usage: java EventQue hostName period(milliseconds) TotalTime(seconds)
 
 
S.No Arguments Description Values
1 hostName Name of the machine where NMS is running String
2 period The time period in which events are buffered int (milliseconds)
3 TotalTime The total time interval after which all the buffered events are dumped into files int (seconds)

back
