[Fujitsu]: How do you map Schema to POJO?
[AdventNet]: Java objects that are created have to be persisted in a data store so that these objects and their associated properties can be retrieved for future reference and manipulated with ease. Web NMS Persistence Services allow Java objects to be stored in a relational database, maintain the relationship between these objects and provide a mapping between these objects and their relational counterparts. This service uses Hibernate, a powerful, high performance object/relational persistence and query service to provide Object-to-Relational mapping.

And we provide a Eclipse plugin (the documentation for which is available) to model your custom objects with minimal efforts (once the object is modeled, the relational sources are DB configuration entries are generated)
